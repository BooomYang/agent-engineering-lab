# Operating Loop

## Weekly Or Per-Source Loop

1. Intake：登记新资料或实践。
2. Triage：判断它属于哪个 agent engineering surface。
3. Extract：拆成事实、观点、推断、假设。
4. Map：更新方法论、OpenAI 实践、模板、案例或实验。
5. Validate：通过真实项目、trace 或 eval 验证。
6. Consolidate：反复出现的问题升级为 checklist、eval、skill 或 framework backlog。

## Update Decision

| Input | Where it goes first |
| --- | --- |
| OpenAI 官方文档 | `01_sources/sources.md` and `03_openai_agent_practice/` |
| 第三方文章 | `01_sources/article-note-template.md` |
| 一次实践失败 | `04_templates/practice-retro.md` |
| 完整项目复盘 | `05_cases/` |
| 待验证判断 | `06_experiments/` |
| 反复出现的流程 | `04_templates/` or `07_framework_roadmap/` |

## Quality Bar

一次更新完成时，至少满足：

- 来源有登记。
- 结论有归属面。
- 不确定性被标注。
- 能复用的内容进入模板或 checklist。
- 和 OpenAI 产品能力有关的主张有官方链接支撑。
