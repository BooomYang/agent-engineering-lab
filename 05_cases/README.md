# Cases

这里存放真实 agent 项目案例和复盘。

## Case Naming

建议格式：

```text
YYYY-MM-DD-project-name-scenario.md
```

## Case Structure

每个案例至少包含：

- 背景：做什么 agent，服务什么用户目标。
- 架构：agent definition、tools、state、orchestration、guardrails、evals。
- 过程：关键设计决策和遇到的问题。
- 证据：trace、日志、截图、用户反馈、eval 结果。
- 复盘：哪些方法论被验证，哪些需要修正。
- 资产：沉淀出的模板、checklist、eval 或 backlog。

## Intake

如果只是一次小实践，先用 [practice-retro.md](../04_templates/practice-retro.md)。当实践足够完整，再整理成 case。
